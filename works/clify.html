💡

아래 내용은 기획 당시의 의도를 정확히 보여드리고자 일부 재구성하여 작성하였습니다.
당시 시장 상황도 함께 작성하였으니, 당시의 상황도 고려하여 읽어주시기 바랍니다.

기존 웹사이트의 문제점

  • 기존에 클라이피에서 운영하던 웹사이트는 유저 친화적이지 않음
    • 상담사 검색이 매우 불친절하고 불편함
    • 웹사이트 내에서 유저 액션을 유도할 장치가 없음
    • 실제로 입사 후 단 한 건의 구매도 일어나지 않음
  • 웹사이트에 등록된 상담사에게도 불편한 시스템
    • 별도의 상담 진행 시스템 없이 카카오톡, 줌 등 상담사의 계정으로 상담이 진행됨
    • 상담 요청 횟수도 적거나 없고, 이에 상담 중지를 요청하였으나 제대로 관리도 되지 않음

회사 내부의 문제

  • 기존 웹 사이트를 대체할 앱 서비스 필요
    • 상담을 받는 사람들 대다수가 편하게 사용할 수 있는 스마트폰 앱 필요
    • 단순 B2C용이 아닌 EAP 용 B2B 유저도 함께 이용할 수 있어야 함
  • 기존 자원 활용 불가
    • 웹사이트 구축을 외주 업체에서 진행하였으며, 제작 완료 후 계약이 종료됨
    • 해당 업체와 소통하던 담당자가 이미 퇴사하였으며, 인수인계 자료가 미흡

시장 상황

  • 중대재해처벌법 강화, 정부의 지원(2조 이상의 전국민 마음투자산업) 등으로 국내 멘탈 웰니스 산업은 계속해서 성장할 것으로 기대

🧭 기획 목적 및 목표

기획 목적

  • 기존 웹사이트를 완전히 대체할 수 있는 앱을 개발한다.
  • 유저와 상담사가 편리하고 신뢰할 수 있는 서비스를 개발한다.

기획 목표

  • 유저 맞춤 상담사 추천 기능 추가
  • 유저 액션 유도 기능 추가
  • 커뮤니케이션 솔루션 도입으로 상담 편리성 증대
  • 3월 첫 주차에 앱스토어/플레이스토어 런칭

기능정의서, 정책서 작성

  • 탭 별 기능 정의 (Notion)
    • 기능ID 부여, 구분(탭), 세부 구분(탭 내 화면), 기능, 세부기능(기능 동작)으로 뎁스 분류
    • 개발자의 이해를 돕기 위해 기능 설명과 기획 의도를 함께 기술
    • APP, Backend 개발 완료 체크란을 두어 개발 진행 상황 체크
    • 추후 Figma 화면별 ID 부여하여 각 기능이 어떤 화면에 포함되는지 표기
  • 정책서 작성 (Notion)
    • 회원 정책 - 회원의 종류, 일반/기업 회원의 화면별 권한, 탈퇴 및 재가입
    • 상담사 검색 및 예약 정책 - 상담사 및 상담사 관련 정의, 상담사 검색, 상담사 예약, 심리검사 예약
    • 결제 및 환불 정책 - 상담 비용 책정 방식, 포인트 정책, 쿠폰 정책, 결제 정책, 취소 및 환불 정책, 상담사 정산 정책
    • 후기 및 상담사 교체 정책 - 후기 정책, 상담사별 별점 계산 정책, 상담사 교체 정책
    • 기타 정책 - 마음배터리 정책, 앱 권한

와이어프레임 제작

  • 각 화면별 기능 배치
  • 각 화면별 UX Writing

주요 기능 기획

  • 상담사 검색 - 고민으로 검색하기
    • 목적 : 유저가 자신의 고민을 AI 챗봇에게 입력하면, 몇 번의 질문으로 적절한 상담 주제를 도출하여 해당 상담이 가능한 상담사를 추천함
    • 구현 방식
      • 여러 고민 예시를 활용하여 입력 및 출력 토큰의 평균값으로 학습 및 운영 비용을 계산하여 적절한 LLM 모델 제안
      • Open AI의 LLM을 활용하여 프롬프트 엔지니어링과 Fine-tunning으로 학습 모델 개발
  • 미션 및 마음배터리
    • 목적 : 처음 앱에 진입한 유저에게 앱 튜토리얼을 미션 형태로 제공하여 앱에 대한 이해도를 높이고, 미션 완료 후 현재 마음 관리 현황을 배터리로 표현하여 지속적인 액션을 유도함
    • 구현 방식
      • 처음 진입한 유저가 홈 탭에서 다섯가지 미션을 진행할 수 있으며, 각 미션은 소요 시간 기준으로 30초 이내에서 50분까지 점점 늘어나는 형태로 구성
      • 쉬운 미션을 클리어한 유저가 미션을 모두 달성하기 위해 앱의 메인 기능인 심리검사와 심리상담까지 경험할 수 있게 유도함
      • 모든 미션을 달성한 유저는 마음 관리 현황을 보여주는 '마음배터리'를 보여주어, 앱 내 리텐션을 유지하도록 설계
  • 커뮤니케이션 솔루션 도입
    • 목적 : 채팅 상담, 음성 상담, 화상 상담을 하나의 솔루션으로 활용할 수 있게 하여, 기존 상담사들이 여러 솔루션에서 개인 계정을 활용하던 불편함을 해소함
    • 구현 방식
      • 샌드버드, 줌 등 여러 솔루션을 비교하여 최종적으로 채팅, 음성, 화상 커뮤니케이션이 모두 가능한 샌드버드 SDK를 사용하는 것으로 확정
      • 솔루션 사용 방법을 매뉴얼로 제작하여 상담사에게 배포
  • 기타 API 도입
    • 목적 : 여러 솔루션 업체를 비교 분석 후 적용하여 가장 합리적인 비용으로 빠른 개발 진행
    • 구현 방식
      • 본인인증 - 다날
      • 결제 - 나이스페이먼츠
      • 알림톡 - NHN 클라우드

QA 후 스토어 런칭

  • QA 시트 작성 (Notion)
    • 기능정의서에서 부여한 기능ID를 기준으로 설명된 기능 동작 여부를 체크할 수 있게 구성
    • 버그 발생 시, 티켓 생성 버튼을 누르면 각 개발자 Notion 페이지에 티켓이 생성되고, 해당 프로젝트 Slack 채널에 알림이 가도록 설정
    • 각 티켓 별 개발 상태도 QA 시트에 함께 표시하여, 개발자가 티켓 상태를 개발 완료로 설정 시 바로 시트에 반영되도록 구성
  • 스토어 정보 작성
    • 각 개발자 콘솔 내에서 필요한 앱 기본 설정 세팅
    • 앱 이름, 상세 정보 및 검색어 설정
    • 스토어 미리보기 이미지 제작 (Figma)
  • 기타 런칭 관련 업무 진행
    • 사전예약 페이지 제작 및 런칭 후 사전예약 대상자에게 안내 메일 발송
    • 상담사 정보 업데이트 및 관련 매뉴얼 제작하여 전달
