고객이 직면한 문제

  • 스터디그룹 기능이 추가된 이후 4달 동안 다른 유저와 소통이 VOC 중 두 번째로 많은 수로 들어옴 (120 건 중 11건)
    • 실제 유저 VOC 일부
      • 스터디그룹에서 멤버들과 소통할 수 있게 해주세요.
      • 친구추가 기능과, 메시지 기능, 오답문제를 다시 풀 수 있게 해주세요!!!!!😭😭😭
      • 스터디 그룹에서 대화를 하고싶어요!
      • 스터디그룹에서 간단한 채팅 기능과 문제 질문 기능을 넣어 주세요
      • 친구와 함께 할 수 있게 해주세요 친구 맺기같은걸 해서 어디까지 진도가 나갔는지 정답률은 어느정도인지 등을 볼 수 있게 해서요
      • 다른 에그릿과 친구맺기하게 해주세요 서로 응원하고 같이 진도를 나가거나 할 수도 있게요
      • 스터디룸에서 질문이나 간단한 채팅을 할 수 있으면 좋겠어요!

회사 내부의 문제

  • 스터디그룹 서비스를 런칭한 후 3개월 이상 소통 관련 업데이트가 없었음
  • 시험 직후 유저 DAU가 떨어지고 있어서 DAU 감소를 방어할 수 있는 신규 기능이 필요함

🧭 기획 목적 및 목표

기획 목적

  • 유저가 학습 외에도 앱에 접속할 수 있는 서비스를 만들어서 시험 기간이 아니더라도 DAU가 유지될 수 있게 한다.

기획 목표

  • 2학기 중간고사 직후(10월 초) 1~2주 동안 시험 기간 75% 정도 DAU 유지를 목표로 함
    • 관련 수치
      • 평시(방학 및 학기 중) DAU는 시험 기간 대비 75% 정도
      • 시험 직후 1~2주 DAU는 시험 기간 대비 60% 정도

🏗 서비스 제작 과정

기술 검토

  1. 웹소켓을 활용한 실시간 통신
    • 채팅방에 입장한 상태에서 웹소켓 통신으로 실시간 메시지 전송 및 확인 가능
    • 채팅방 입장 전 가장 최신 메시지 미리보기 지원 가능
  2. 마지막으로 읽은 메시지 확인
    • 마지막으로 읽은 메시지를 저장할 수는 있으나 채팅방 입장 시 마지막 읽은 메시지와 최신 메시지 사이 내용을 로딩하는데 오래 걸릴 여지가 있어 최초 개발에서 제외
  3. 그 외 부가 기능
    • 메시지 복사 기능은 디바이스 자체 기능으로 사용
    • 메시지 삭제 기능은 아래와 같이 분기를 나눠 사용하며, 카카오톡과 같이 삭제한 후에도 작성한 내역은 확인할 수 있음
      • 스터디그룹장 : 모든 메시지 삭제 가능
      • 일반 유저 : 내가 보낸 메시지만 삭제 가능
    • 메시지 신고 기능은 기존 답변 신고와 동일한 flow로 사용 → 신고 처리를 위한 BOS 필요
    • 메시지 답장 기능은 원문과 답문 사이 메시지를 로딩하는데 오래 걸릴 여지가 있어 최초 개발에서 제외
    • 링크 전송은 타 메신저와 동일하게 링크와 썸네일을 함께 표시할 수 있음 (추후 앱 내 질문을 채팅방에 공유하는 기능을 추가하기 위해 필요한 기능)

실시간 채팅 기능 제작

  1. 유저 flowchart 제작
    • 채팅방 입장부터 채팅 입력 및 채팅 부가 기능 이용 관련 flowchart 제작

개형 — 유저 플로우차트

  1. 화면 기획 및 제작
    • 채팅 방 입장 버튼 배치
      • 기존 스터디그룹 홈에 배치한 그룹원 정보 버튼을 채팅방 입장 버튼으로 교체
      • 그룹원 정보는 스터디 그룹 정보 화면에서 확인할 수 있도록 변경
      • 아래 세 가지 케이스로 버튼에 정보 표시
        • 아무도 채팅을 보내지 않음 : '메시지를 보내 채팅을 시작해봐요!' 문구 표시
        • 채팅 내역이 존재하며, 모두 확인함 : 가장 최근에 전송된 메시지 표시
        • 확인하지 않은 채팅 내역이 존재 : 가장 최근에 전송된 메시지와 함께 확인하지 않은 채팅 수 표시
    • 채팅방 내 기능 활용 관련 부가 기능
      • 채팅 내용을 1초 이상 탭하면 바텀 모달로 아래 기능 사용 가능
        • 복사하기 (모든 채팅 내용에 가능)
        • 신고하기 (내가 보낸 채팅 외 내용에만 가능)
        • 삭제하기 (내가 보낸 채팅이거나 그룹장이 탭한 때에만 가능)
      • 채팅 내용이 너무 길 때 정책
        • 채팅은 최대 1,000자까지 작성 가능
        • 모바일 기준 20줄, 태블릿 기준 6줄만 우선 표시되고 나머지 내용은 '전체보기'를 눌러 확인할 수 있음
    • 기타 정책
      • 스터디그룹 입장 시 무조건 최신 메시지 기준으로 내용 표시
      • 스터디그룹 입장 전 발송된 채팅 내역도 확인 가능
      • 분 단위로 동일한 시간에 한 유저가 여러 채팅을 보낼 때, 프로필 사진은 해당 시간에 첫 메시지에, 전송 시간은 가장 최신 메시지에만 표시
      • 그룹원이 채팅창에 존재하는 도중 그룹에서 강퇴 당하면, 채팅 전송 버튼 클릭 시 오류 토스트 메시지가 표시되고 내용이 전송되지 않음

신고 BOS 제작

  1. 관리자 flowchart 제작
    • 유저 신고 후 신고 내용 처리까지 flow 정리

개형 — 관리자 플로우차트

  1. 화면 기획 및 제작
    • 신고 내역 검색 필터 설정
      • 유저 ID로 검색
        • 피신고유저 : 신고된 유저의 히스토리를 모아볼 수 있게 함
        • 신고유저 : 악의적인 허위 신고를 하는 유저를 필터링하기 위함
      • 신고 타입 : 신고 대상인 콘텐츠 필터링
      • 신고 사유 : 유저가 신고한 이유로 필터링
      • 상태 : 대기, 경고처리, 반려 등 처리 상태로 필터링
    • 신고 내역 검토 팝업 화면 기획
      • 기본 정보 표시 (피신고/신고 유저, 총 신고 횟수, 마지막 이용 정지 일자, 신고 사유, 신고 내용)
      • 신고 히스토리 : 과거 내역을 보고 처벌 수위에 참고하기 위함 (반려된 내역, 허위/중복 신고 내역 제외하고 확인 가능)
      • 처벌 여부 및 담당자 의견 입력
    • 기타 정책
      • 신고 이후에 삭제된 콘텐츠도 신고 내용은 확인할 수 있어야 함
      • 필기 내역을 신고하면 필기 ID를 표시하여 앱 내 오퍼레이터 메뉴를 통해 확인
      • 신고를 처리한 관리자를 담당자로 표기

목표 달성 여부

  • 채팅 사용률이 기대한 것에 훨씬 못 미치는 수치를 기록함
    • 원인은 DAU에 비해 낮은 스터디그룹 이용률 (약 5~7% 내외)
    • 반면 스터디그룹 이용 수 대비 채팅방 진입 비율은 70%대를 유지
    • 채팅 기능은 스터디그룹에 진입해야만 이용할 수 있기 때문에 스터디그룹 이용율이 높아져야 채팅 사용률도 함께 올릴 수 있음
  • 이후 스터디그룹과 채팅 이용률을 높이기 위한 후속 조치 계획
    • 스터디그룹을 사용해야 하는 필요성 부여
      • 스터디그룹 채팅방에 질문 공유 기능 추가
      • 동일한 교재를 사용하는 유저들을 한 스터디그룹으로 구성
    • 스터디그룹 사용성 개선
      • 스터디그룹 검색 기능 개선
      • 스터디그룹 이용 목적에 맞는 정렬 방식 추가
