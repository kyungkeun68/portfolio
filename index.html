How I Work

기획서를 던지지 않습니다. 데이터와 코드 안에서 함께 결정합니다.

01

데이터로 본다 — 감으로 결정하지 않는다

"정답 입력하다 틀린다", "y/z가 안 보인다" 같은 VOC가 반복되는 것을 발견. "풀이 흐름의 병목이 정답 입력 단계"라는 가설로 좁혀, AI팀의 CV 모델과 협업해 필기 인식 기능으로 검증. 결과는 90개 분석 문항 중 80%에서 풀이 시간 단축, 평균 17%.

필기로 정답 입력 기능 · 개념원리
02

개발자 옆에 앉는다 — 기획서 던지지 않는다

백엔드 리소스가 주 2일밖에 안 되는 상황에서, 기존 API를 큐레이션하는 방식을 함께 설계. 앱 개발자와는 전체 Flow를, 백엔드와는 활용 가능한 엔드포인트를 같이 정리. 내신 대비 동선을 한 달 안에 출시해 주말 DAU +68%, 앱 런칭 후 최대 MAU.

03

AI로 본다 — 모른다고 멈추지 않는다

Claude Code의 skill 기능을 활용해 multi-repo(API/AI/웹앱)를 직접 읽고, MySQL 쿼리도 AI 협업으로 작성. 비개발자가 코드 R&D 사이클을 주도하며 개발자 의존도를 줄이고 가설 검증 속도를 끌어올림.
